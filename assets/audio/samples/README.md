# Sampled-audio overrides

Drop a real audio file here named exactly after a sound id and it will be used
in place of the procedural synth for that sound — no code changes needed. The
synth (`scripts/autoload/sound_synth.gd`) remains the fallback when no file is
present, so the game always has sound.

- Naming: `<id>.ogg`, `<id>.wav`, or `<id>.mp3` (probed in that order).
- Resolved by `AudioBus.synth()` → `_resolve_sample()` (cached after first probe).

## Sound ids currently in use

The full set, generated by `sound_synth.gd`. A file named `<id>.ogg|wav|mp3`
here overrides that id. **Priority drop-ins** (biggest perceived polish jump)
are marked ★.

Weapon fire:
`pistol_fire ★, rifle_fire ★, shotgun_fire ★, plasma_fire ★, gauss_fire,
drone_shot, rocket_fire`

Impacts / hits:
`impact_metal ★, impact_concrete ★, impact_stone, impact_wood, headshot`

Explosions / ordnance:
`explosion ★, grenade_throw, thunder`

Weapon handling:
`empty_click, reload, pump_action, charge`

Player / feedback:
`footstep, player_hurt, breathing, pickup_health, pickup_ammo, combo_up,
victory`

Enemies:
`drone_hum, mech_step, acid_spit, overlord_glitch`

Broadcast / UI:
`eas_alert, broadcast_blip, radio_static`

Music (loop ★ — the single biggest win):
`music_techno ★, music_grok, music_gemini, music_suburb, music_archon,
music_lava, music_water`

Ambience beds (seamless loops):
`ambience_drone, ambience_wind, ambience_rain, lava_loop, water_loop`

Example: add `pistol_fire.ogg` here and every pistol shot uses it; remove it and
the synth takes back over. No code changes either way.

Source CC0 packs that fit (download + drop in): Kenney "Sci-Fi Sounds" /
"Impact Sounds", or freesound.org CC0 clips.
